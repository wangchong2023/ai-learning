# Planner Lab: 深度规划与执行实验场

本模块展示了高级智能体架构中最硬核的部分：**Plan-and-Execute (计划与执行) 模式**。它完美解决了原生 ReAct 架构在面对多步复杂任务时“走一步看一步”的短板。

## 🧩 为什么需要 Plan-and-Execute？
在处理复杂逻辑任务（如“先搜集 A 公司的财报，再搜集 B 公司的财报，最后对比它们的毛利率差距”）时：
1. **原生 ReAct** 很容易在中间某个环节迷失方向或陷入死循环。
2. **Plan-and-Execute** 则像一个成熟的架构师，先全局统筹画出**施工蓝图**，然后再由执行单元按图索骥。

## ⚙️ 核心流转机制
我们利用 LangGraph 拆解了三大独立节点：
1. **战略家 (Planner)**：基于用户的终极目标，使用结构化输出 (Structured Output) 强制生成一张严谨的待办清单 (`List[str]`)。
2. **执行官 (Executor)**：毫无杂念地只关注清单上的**第一步**，将其执行并把结果存入历史记录 `past_steps` 中。
3. **复盘官 (Replanner)**：根据执行官的战报，进行全局反思 (Reflection)。如果目标达成，向用户汇报；如果有偏差或未完成，**动态修改并生成新的剩余计划**。

## 🚀 快速体验
```bash
make plan
```

## 💡 实验建议
* 输入高度复合的指令（如：“分析2023年AI的三个突破，并用这些突破来预测2024年的两个风险”），观察 **[计划生成]** 的结构化拆解。
* 观察 **[计划更新]** 阶段大模型是如何“动态感知”进度并调整待办事项的。
